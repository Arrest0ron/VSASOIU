# Data Path процессорного ядра ASOIU-16

## 1. Что такое Data Path

Data Path - это часть процессора, через которую физически проходят данные. Если ISA описывает, какие команды существуют, то Data Path показывает, какими блоками эти команды выполняются.

В ASOIU-16 Data Path содержит:

- счетчик команд `PC`;
- память команд `instr_mem`;
- регистр текущей команды `instr_reg`;
- регистровый файл `R0`...`R7`;
- ALU;
- память данных `data_mem`;
- внутренние регистры результата ALU и данных памяти.

Управляющий автомат не выполняет вычисления сам. Он только включает управляющие сигналы: когда читать команду, когда писать регистр, когда писать память, когда обновлять `PC`.

## 2. Упрощенная структурная схема

```text
                   +----------------+
                   | Control Unit   |
                   | FSM            |
                   +--+---+---+---+-+
                      |   |   |   |
                      |   |   |   +-------- mem_write
                      |   |   +------------ reg_write
                      |   +---------------- pc_write
                      +-------------------- alu_op

 +------+     +-------------+     +----------------+
 |  PC  +---->| instr_mem   +---->| instr_reg      |
 +--+---+     +-------------+     +-------+--------+
    |                                      |
    |                                      v
    |                              поля opcode/rd/rs
    |
    |         +----------------+     +-------------+
    +<--------+ PC update mux  |     | Register    |
              +----------------+     | File R0-R7  |
                                     +------+------+ 
                                            |    |
                                            v    v
                                         +---------+
                                         |  ALU    |
                                         +----+----+
                                              |
                                              v
                                      +---------------+
                                      | ALU result    |
                                      +-------+-------+
                                              |
                          +-------------------+-------------------+
                          |                                       |
                          v                                       v
                    +------------+                         +-------------+
                    | data_mem   |                         | writeback   |
                    +------------+                         | to register |
                                                           +-------------+
```

## 3. Как проходит команда ADD

Команда:

```asm
ADD R3, R1, R2
```

Поток данных:

1. `PC` указывает адрес команды.
2. `instr_mem[PC]` выдает команду.
3. Команда сохраняется в `instr_reg`.
4. Из команды берутся поля `rd = R3`, `rs1 = R1`, `rs2 = R2`.
5. Регистровый файл выдает значения `R1` и `R2`.
6. ALU выполняет сложение.
7. Результат ALU записывается в `R3`.
8. `PC` увеличивается на 1.

## 4. Как проходит команда LOAD

Команда:

```asm
LOAD R4, [R0 + 10]
```

Поток данных:

1. Команда загружается из памяти команд.
2. Из регистрового файла читается `R0`.
3. Immediate `10` расширяется до 16 бит.
4. ALU вычисляет адрес `R0 + 10`.
5. Из `data_mem[10]` читается слово.
6. Прочитанное слово записывается в `R4`.
7. `PC` увеличивается на 1.

## 5. Как проходит команда STORE

Команда:

```asm
STORE R3, [R0 + 10]
```

Поток данных:

1. Из регистрового файла читаются `R0` и `R3`.
2. ALU вычисляет адрес `R0 + 10`.
3. Значение из `R3` записывается в `data_mem[10]`.
4. `PC` увеличивается на 1.

## 6. Как проходит команда BEQ

Команда:

```asm
BEQ R3, R4, 2
```

Поток данных:

1. Регистровый файл выдает значения `R3` и `R4`.
2. Data Path сравнивает эти значения.
3. Если они равны, `PC` получает `PC + 2`.
4. Если они не равны, `PC` получает `PC + 1`.

## 7. Связь Data Path и Control Unit

Data Path хранит и обрабатывает данные. Control Unit управляет моментами, когда эти действия разрешены.

Например, для `ADD` управляющий автомат включает:

- `ir_write` в состоянии `FETCH`;
- нужный `alu_op` в состоянии `EXECUTE`;
- `reg_write` в состоянии `WRITEBACK`;
- `pc_write` для перехода к следующей команде.

Для `STORE` он включает:

- вычисление адреса через ALU;
- `mem_write` в состоянии `MEMORY`;
- `pc_write` после записи в память.
